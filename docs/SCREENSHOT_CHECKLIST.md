# Screenshot Checklist

Use this checklist when capturing README, investor, and demo screenshots.

## Required Screens

- [ ] System status / overview
- [ ] Upload Data before file selection
- [ ] Upload Data with validated file
- [ ] Upload Data while worker is queued or starting
- [ ] Upload Data after processing completes
- [ ] Findings / operator report
- [ ] Supporting evidence section
- [ ] Evidence Replay
- [ ] Replay controls
- [ ] Observability / readiness surface
- [ ] Empty state with no uploaded data
- [ ] Error state for failed upload or unavailable backend

## Device Sizes

Capture each critical flow at:

- [ ] Mobile portrait
- [ ] Tablet portrait
- [ ] Tablet landscape
- [ ] Desktop browser

## Quality Rules

- Use current UI only.
- Do not show secrets, tokens, internal access codes, private customer data, or real facility names unless explicitly approved.
- Prefer generic system/facility language over single-industry language.
- Avoid screenshots that imply Neraium is only for cultivation.
- Show evidence and replay, not just dashboards.
- Show at least one useful empty state.
- Show at least one completed upload result.

## README Hero Recommendation

Use one screenshot that shows the system intelligence workflow clearly:

1. Current system status
2. Finding or evidence summary
3. Replay or upload result context

The best hero image should make the product feel operator-ready and evidence-backed within five seconds.
